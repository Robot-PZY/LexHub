# LexHub UI Design System

本文件是律枢前端的设计基线。新增页面和组件应先复用 `src/components/ui`，再按本文件扩展；禁止在页面中另起一套颜色、圆角或阴影。

## 1. 设计定位

- 产品气质：专业、可信、清晰、可追溯。
- 视觉方向：Swiss Modernism 的理性网格 + 轻量描边卡片 + 法律科技品牌色。
- 展示重点：智能体编排、DAG、工具调用、阶段成果、最终报告和回放。
- 官方 Logo：始终使用 `/brand/lexhub-ui-mark.png`，不得重绘、变形、换色或替换。
- 参考图只借鉴清楚的边框、按钮反馈和信息层级，不采用宠物品牌的高饱和橙色、卡通字体和厚重投影。

## 2. 设计原则

1. 一个页面只保留一个主操作，其他操作降为次级或文字按钮。
2. 先用布局、字号和留白建立层级，再使用颜色。
3. 青绿色只表示运行、成功或当前状态；金色只用于品牌强调；红色只表示错误或高风险。
   浅色区域的页面标题、区块标题和卡片标题统一使用 `--color-primary`；深色区域标题使用白色。
4. 同一层级的卡片必须使用相同边框、圆角和阴影。
5. 不使用装饰性大渐变、AI 紫粉渐变、无意义玻璃拟态和持续动画。
6. 空状态、加载状态、错误状态都必须提供文字说明和下一步操作。

## 3. 颜色

颜色以 `src/styles/tokens.css` 为唯一来源。

| 语义 | Token | 用途 |
| --- | --- | --- |
| 主色 | `--color-primary` | 主按钮、标题、导航强调 |
| 运行/成功 | `--color-secondary` | 当前步骤、成功状态、智能体在线 |
| 品牌点缀 | `--color-accent` | Logo 呼应、极少量分隔线和徽章 |
| 页面背景 | `--color-bg` | 全局画布 |
| 卡片背景 | `--color-surface` | 表单、面板、表格 |
| 主文字 | `--color-text-strong` | 标题和关键数据 |
| 正文 | `--color-text` | 正文内容 |
| 次要文字 | `--color-muted` | 辅助说明 |
| 边框 | `--color-line` | 卡片、输入框、表格分隔 |

禁止在 JSX 中新增十六进制颜色。特殊深色展示区域也应优先使用 `--editorial-navy`。

## 4. 字体与层级

- UI 字体：`--font-sans`，中文使用系统无衬线字体，避免网络字体阻塞。
- 页面 H1：30–40px，700；移动端 28–32px。
- 首页区块 H2：26–32px，700；应用页面区块 H2：20–24px，650–700。
- 卡片标题：15–18px，600–700。
- 正文：14–16px，行高 1.6–1.75。
- 辅助文字最小 12px；10–11px 只用于眉题、序号和状态元数据。
- 单行正文最大宽度约 72 个字符。

## 5. 网格与间距

- 以 4px/8px 为间距基准，优先使用 token：`--space-2/3/4/5/6/8/10/12`。
- 用户端内容最大宽度 1360px；事项受理最大宽度 1240px；管理端最大宽度 1480px。
- 页面横向内边距：桌面 28–42px，平板 18–24px，手机 14–16px。
- 区块间距：24px；强分区 40–56px；卡片内部 16–24px。
- 所有 Grid/Flex 子项必须设置 `min-width: 0`，避免中文被挤成竖排。

## 6. 组件规范（项目 UI Kit）

优先使用 `src/components/ui`：

- `Button` / `IconButton`：默认高度 42px，主按钮 48px，图标统一使用 Lucide。
- `Panel`：白色表面、1px 边框、10–16px 圆角；仅重要交付卡使用中等阴影。
- `Badge` / `Status`：状态必须包含文字，不得只依赖颜色。
- `TextField` / `Textarea`：始终显示 label；错误紧邻字段展示。
- `PageHeader`：标题、说明、状态和操作保持同一结构。
- `EmptyState` / `LoadingState`：说明当前状态，并提供可执行动作。
- `StatCard`：数字使用等宽数字，最多一项强调色。

页面不得重复实现 Button、Badge、Panel 的基础样式。新模式成熟后先抽入 UI Kit，再扩展到其他页面。

## 7. 边框、圆角与阴影

- 普通边框：1px `--color-line`。
- 聚焦边框：2px `--color-secondary`，外偏移 2–3px。
- 小控件圆角：8–10px；卡片：12–16px；大型首页容器：20–28px。
- 普通卡片使用 `--shadow-xs`，浮层使用 `--shadow-sm`，关键展示最多使用 `--shadow-md`。
- 不使用黑色硬投影复制参考图；可以用 2px 品牌色底边或细强调线表达层级。

## 8. 页面结构

### 首页

平直导航 → 左文案/右产品界面的分栏首屏 → 数据带 → 产品能力 → 深色智能体执行区 → 办理流程 → 解决方案与回放 → 任务路径 → 会员方案。首页强调真实产品界面，不使用居中巨型口号、重复 Logo 或全页大圆角容器。

- 边框只设三级：弱边框用于内容分组，标准边框用于控件，强边框仅用于产品预览和选中状态；不为每张卡片叠加粗描边。
- 每个区块只保留一个主操作。主按钮使用品牌深蓝实底，次操作使用白底细边框；悬停与按压反馈不得造成布局抖动。
- 阴影只用于产品预览、主操作和推荐方案。普通内容卡默认保持平面，交互时才轻微抬升。
- 首页区块不能重复使用同一种“白底 + 细边框 + 圆角卡片”构图。产品能力使用品牌色块，办理流程使用轴线，解决方案使用编辑式分隔线，历史回放使用深色状态面板。
- 品牌几何语言来自 Logo 的纵横轴：青绿表示运行和连接，金色表示节点与重点，深蓝表示控制、归档与可信交付。
- 同一信息只展示一次；“办理流程”和“任务路径”不得作为两个近似区块同时出现。

### 事项受理

宏观阶段导航 → 横向微步骤 → 简短说明 → 办理类型 → 事项名称与需求 → 材料 → 单一主提交按钮。禁止再使用左侧嵌套步骤栏。

### 执行工作台

顶部运行遥测 → 左侧阶段路径 → 中央 DAG/阶段结果 → 右侧工具调用轨迹。工具名称、调用顺序、状态和结构化结果必须可见。

### 结果交付

完成状态 → 交付目录 → 总结报告 → 场景专项结果 → 文档下载 → 依据与工具附录。

### 管理端

- 用户端使用明亮任务工作台：白色侧栏、浅灰画布、紧凑页头；主视线优先落在事项状态、材料和下一步操作。
- 管理端保留深色控制台侧栏，内容区采用高密度浅色数据面板；表格、配置卡和状态标签避免装饰性大阴影。
- 两端共享 44px 控件高度、三级边框、9–12px 控件/卡片圆角，以及一致的 hover、pressed、focus-visible 状态。
- 数据页优先使用表格、分段导航和低阴影分组；深色渐变仅用于系统总览等重点状态区域。

固定侧栏 → 顶部面包屑 → 页面标题 → 状态/统计 → 配置卡片或表格。管理端密度可高于用户端，但不得缩小到难以阅读。

## 9. 响应式断点

- `> 1200px`：固定侧栏，多列工作台。
- `769–1200px`：侧栏抽屉化，主内容全宽，复杂三列降为两列/单列。
- `<= 768px`：单列；横向阶段条可滚动；次要状态收起。
- `<= 560px`：紧凑页面内边距，按钮允许整行，表格转卡片或横向滚动。

同一组件的“隐藏侧栏”和“取消主区偏移”必须使用相同断点。

## 10. 动效与可访问性

- 交互动画 140–220ms，仅使用 `opacity` 和 `transform`。
- 支持 `prefers-reduced-motion`，不能让动画阻塞操作。
- 所有可点击目标建议不小于 44×44px。
- 图标按钮必须有 `aria-label`；页面必须有可见键盘焦点。
- 普通正文对比度至少 4.5:1。

## 11. 文案规范

- 导航名称与对应模块标题保持一致。
- 标题直接说明内容；说明文字只写用户可以做什么。
- 除产品名、数据格式和必要缩写外，不使用英文眉题或英文口号。
- 避免“不是……而是……”“赋能”“闭环”“沿路径”“持续展示”等抽象表达。
- 优先使用“上传、查看、生成、下载、回放”等明确动作。
- 同一信息只解释一次；能用一行说清的内容不写成两段。

## 12. 禁止事项

- 禁止复制参考网站的品牌、插画和配色。
- 禁止同时使用三种以上强调色。
- 禁止页面级 inline style 定义新的视觉规范。
- 禁止通过不断追加高优先级覆盖修复结构问题；应删除冲突规则或将页面迁移到对应的 `*-v2.css`。
- 禁止修改 `Co-Sight-master-raw`。
