# 阶段 5：自动核验与可靠收口

## 问题

旧链路只有全部步骤标记 `completed` 才发送完成信号，并在所有步骤后额外调用一次模型生成总结。外部搜索超时、智能体线程异常或最后一次模型调用缓慢时，会出现“过程已经展示，但任务一直不结束、最终结果为空”。

## 收口策略

- 单步骤默认执行预算为 45 秒，避免演示链路被单个外部服务长期占用。
- 依赖步骤只有真正完成才放行；失败依赖不会继续制造级联错误。
- 线程异常会把当前步骤标记为 `blocked` 并发布阶段事件。
- 执行结束后把剩余非终态步骤收敛为带原因的 `blocked`。
- 最终报告确定性汇总各专业智能体已经形成的阶段结论，不再依赖额外一次模型请求。
- 自动核验检查步骤终态、阶段结论、可追溯来源和交付产物，输出评分及 `passed` / `passed_with_warnings`。
- WebSocket 在“全部步骤终态且结果非空”时发送完成信号；存在降级步骤时返回 `finished_with_warnings`，前端仍进入结果页。

这样既保留模型在专业分析阶段的价值，也保证评委演示能够一步一步走到真实最终结果。
