# 阶段 4：统一执行事件与结果展示

## 统一表达

所有能力调用统一使用 `CapabilityResult`：状态、摘要、结果类型、业务数据、来源、产物、指标、错误和降级信息。旧工具仍可运行，但其事件会补充 `agent_id`、`capability_id` 和结构化能力结果，前端无需针对每家 API 编写一套运行记录组件。

## 前端层级

1. 运行记录概览：显示实际智能体、能力 ID、结果类型、来源数与产物数。
2. DAG 步骤详情：显示专业智能体、阶段能力、并行组、执行条件和预期产物。
3. 工具结构化结果：按需展开来源链接、生成产物和业务数据，避免一次性堆满评委视野。
4. 回放与实时执行复用相同的 `ToolCallTrace` 和 `StepToolDetail`，保证展示口径一致。

该设计遵循渐进披露与状态可见性原则：评委首先看到“谁在何阶段调用了什么并取得何种结果”，需要时才展开原始结构化数据。
