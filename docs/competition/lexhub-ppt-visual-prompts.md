# 律枢 LexHub PPT 配图与图表生成提示词

用途：给 GPT、PPT 外包或设计工具生成正文页图形、图表和截图包装。  
说明：封面和封底已有底板，不在本文档中生成。  
结构：与 `lexhub-ppt-outsourcing-plan.md` 保持一致，正文分五章、十三页。

## 0. 通用提示词

每次生成图表前，可复制以下通用约束。

```text
请为 16:9 国赛答辩 PPT 设计一张可编辑的信息图或截图包装页。

项目是“律枢 LexHub：基于 Co-Sight 的法律行业超级智能体工作台”。

视觉风格：
科技竞赛风，法律行业稳重感，浅色背景，深蓝 #123A63，青蓝 #1C8CA8，金色 #D99A2B 少量强调。

设计要求：
1. 使用卡片、表格、流程箭头、线性图标、细线框。
2. 不要生成复杂背景图，不要使用人物照片、法槌大图、夸张 3D。
3. 中文文字必须准确、简洁、清晰。
4. 图形应方便在 PowerPoint 中复刻，尽量使用基础形状。
5. 不要编造性能百分比、市场规模、用户数量。
6. 项目已有数据只能使用：9 类业务场景、6 个法律智能体、5 类工具能力、4 层系统架构、2 个核心 JSON 配置。
```

截图页通用约束：

```text
我会提供真实系统截图。请只做 PPT 展示包装，不要重绘界面，不要伪造不存在的菜单、按钮、数据。

请处理为：
1. 保留截图主体，外加轻量浏览器窗口边框。
2. 对关键区域加 3 到 4 个编号标注。
3. 右侧、侧边栏或局部放大框放简洁说明。
4. 使用深蓝、青蓝、金色作为标注色。
5. 可以裁切、放大重点区域、加阴影，但不能改截图内容。
6. 如果截图中没有某功能点，不要在说明里写该功能。
```

## 第一章 业务场景

### 第 1 页：项目背景

图形类型：背景动因 + 传统方式到 LexHub 方案转化图

```text
请生成一张“LexHub 项目背景”信息图，风格参考科技竞赛 PPT 的开场背景页。

画面结构：
左侧 35% 放设计动因：
法律业务不是一次问答，而是多材料、多依据、多交付物的复杂任务。

右侧 65% 放“传统方式 → LexHub 方案”的转化关系：
传统方式：
人工拆解任务、人工检索法规、人工整理材料、人工撰写文书、人工复核过程

LexHub 方案：
智能编排、法规增强、材料关联、文书生成、过程回放

右下角放三枚痛点徽章：
1. 效率低：合同审查、争议整理、法规研究耗时长
2. 过程不透明：任务过程难复盘、难追责、难审核
3. 交付成本高：报告、函件、合同草稿整理依赖人工

页面结论：
法律任务需要可编排、可追溯、可复核。

设计要求：
转化关系使用左右对照或上下递进结构，中间用箭头连接。
痛点用右下角徽章或侧边提示条呈现，金色编号突出。
不要使用法槌、人物、法院照片。
```

```text
底边用小标签展示 9 类场景：

设计要求：
```


### 第 8 页：DAG 编排

图形类型：条件触发 DAG 图 + 工具能力标签

```text

法规研究 → 文书生成 → 交叉审查 → 合规监测 → 结果交付

材料不足、引用不足、需要文书、高风险导出

右侧竖排工具能力标签：
联网搜索、代码执行、文档/OCR、法规检索、DOCX/PDF 导出

设计要求：
```
