# 律枢 LexHub 创新总结报告（国赛必交）

## 一、方案概述

律枢 LexHub 是基于 **中兴 Co-Sight 超级智能体框架** 二次开发的法律业务智能服务平台，面向合同审查、争议解决、合规研究等场景，实现「任务理解 → 证据质检 → 法规研究 → 文书生成 → 交叉审查」的可编排、可工具化、可回放闭环。

---

## 二、创新点（≤5 条）

### 1. 状态驱动多智能体，而非固定法律流水线

将法律任务从「固定步骤表单」升级为 **按材料完整度、引用覆盖率、风险等级动态调度** 的多智能体协作，更贴近真实办案逻辑。

### 2. Co-Sight DAG 可视化 + 工作流配置双轨

`legal-workflow.json` 描述编排规则，前端 DAG 执行视图展示 **真实 WebSocket 执行路径**，评委可同时看到「设计」与「运行」。

### 3. 真实执行记录驱动的文书导出

导出 DOCX/PDF 优先写入 **replay / 实时执行快照**（阶段、工具、结论、溯源），而非静态 Demo 模板，强化「可信可复核」。

### 4. 赛题能力一站式演示板（/board）

将赛题硬性要求映射为可点击演示入口，并内置 **3 分钟评委路线** 与 **提交就绪度**，降低答辩现场迷路风险。

### 5. 可复制的 SaaS 商业化路径（规划）

个人三档（体验/Pro/Ultra）+ 机构定制；同一 Co-Sight 底座通过 **工作流包 + 工具包 + 知识库包** 复制到多场景，满足赛题「可复制、可扩展」要求。

---

## 三、技术难点与解决方案

| 难点 | 解决方案 |
|------|----------|
| Co-Sight 执行过程难展示 | WebSocket 事件适配 + DAG/智能体/工具链三面板 |
| 材料上传与回放 CORS | Vite 代理 + 统一 upload 目录 |
| 导出需可复核 | execution-snapshot 解析 replay.json |
| 管理端与用户端叙事分裂 | 独立 Admin Shell + 工作流只读预览 |
| 比赛材料分散 | `docs/competition/` 统一交付 |

---

## 四、推广应用前景

1. **个人律师**：Pro 档覆盖标准办案闭环
2. **高频复杂案件**：Ultra 档深度工具与更高用量（规划）
3. **律所 / 企业法务**：定制私有化 + 行业工作流包
4. **法律科技 OEM**：Co-Sight 二开白标输出

**社会价值**：降低法律辅助工作门槛，保留人工复核与过程审计，避免「黑盒 AI 法律建议」。

---

## 五、团队协作经验（模板，请按实际填写）

- 产品叙事与赛题对齐：统一 Co-Sight 话术，避免「普通法律 SaaS」印象
- 前后端分工：前端演示体验 + 后端 replay/导出 API
- 答辩准备：固定 3 分钟路线 + `/board` 能力对照表
- 文档与代码同步：配置 JSON 与页面展示保持一致
