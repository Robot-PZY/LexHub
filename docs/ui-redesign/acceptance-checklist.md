# UI 改版验收清单

## 基础质量

- [ ] 无中文乱码。
- [ ] 无明显文本溢出。
- [ ] 无按钮文字挤压。
- [ ] 无组件重叠。
- [ ] 页面主操作清晰。
- [ ] 页面标题、说明、状态一致。
- [ ] 所有核心按钮有 hover、focus、disabled 状态。

## 设计系统

- [ ] Figma token 与 CSS token 对齐。
- [ ] 基础组件有 Figma variants。
- [ ] React 组件 props 与 Figma variants 对齐。
- [ ] Button、Input、Badge、Panel、Table 等核心组件统一。
- [ ] 页面不再大量新增一次性样式。

## 用户端主流程

- [ ] 登录成功。
- [ ] 新建事项成功。
- [ ] 选择业务场景成功。
- [ ] 材料上传入口清晰。
- [ ] 提交办理成功。
- [ ] 办理过程可见。
- [ ] 审查结论可读。
- [ ] 法律依据和证据链可见。
- [ ] 文书或报告导出入口清晰。
- [ ] 回放归档入口清晰。

## 管理端

- [ ] 总览页能判断系统状态。
- [ ] 知识库页面能搜索、浏览、导入。
- [ ] 模型配置页面分组清晰。
- [ ] API / 连接状态清晰。
- [ ] 路由规则可理解。
- [ ] 用户和权限管理清晰。
- [ ] 危险操作有确认或明显提示。

## 响应式

- [ ] 1440px 桌面可用。
- [ ] 1280px 桌面可用。
- [ ] 1024px 平板可用。
- [ ] 768px 平板可用。
- [ ] 390px 手机可用。

## 可访问性

- [ ] 表单 label 完整。
- [ ] 图标按钮有 aria-label。
- [ ] 键盘可聚焦。
- [ ] Focus 样式可见。
- [ ] 文本颜色对比度合理。

## 构建与回归

- [ ] TypeScript 构建通过。
- [ ] lint 通过或已记录既有问题。
- [ ] 主流程手工冒烟通过。
- [ ] 关键页面截图留档。
